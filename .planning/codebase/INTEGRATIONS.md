# External Integrations

**Analysis Date:** 2026-07-20

## APIs & External Services

**AI model services:**
- OpenAI Responses API - Optional agent execution and dispute-arbitration judges.
  - SDK/Client: `openai` via `new OpenAI(...)` in `services/api/src/agent-runtime.ts` and `services/api/src/arbitration.ts`.
  - Auth: `OPENAI_API_KEY`; model/provider behavior is controlled by `ARBITRATOR_PROVIDER`, `ARBITRATOR_MODEL`, `ARBITRATOR_TIMEOUT_MS`, and `ARBITRATOR_FALLBACK` in `services/api/src/arbitration.ts`.
  - Local fallback: `DeterministicAgentProvider` and `DeterministicArbitrator` are selected without the key, so the demo has no external AI dependency.

**Blockchain / EVM:**
- Arc Testnet-compatible RPC - Solidity contract deployment and blockchain-event watching.
  - SDK/Client: `ethers` in `contracts/scripts/deploy_testnet.js`; `viem` in `services/indexer/src/index.ts`; Wagmi/Viem browser configuration in `frontend/src/wagmi.ts`.
  - Auth: deployment uses `PRIVATE_KEY`; RPC URLs use `ARC_RPC_URL` in `contracts/scripts/deploy_testnet.js` and `services/api/src/integrations/circle.ts`, and `RPC_URL` in `services/indexer/src/index.ts`.
  - Contracts: `contracts/src/StreamingVault.sol`, `contracts/src/ReputationRegistry.sol`, and `contracts/src/MockUSDC.sol`; deployed-address metadata is written by `contracts/scripts/deploy_testnet.js`.
- Browser wallet signing - The UI can sign creator task approvals, which the API verifies with Viem's `verifyMessage` in `services/api/src/app.ts`.
  - SDK/Client: `wagmi` / `viem` configured in `frontend/src/wagmi.ts`.
  - Auth: end-user wallet signature; no server-side wallet credential is required for verification.

**Circle payments and wallets:**
- Circle Developer-Controlled Wallets - provisions Arc EOA/SCA wallets and submits guarded contract-execution transactions.
  - SDK/Client: `@circle-fin/developer-controlled-wallets` in `services/api/src/integrations/circle.ts` and `services/api/src/integrations/paymaster.ts`.
  - Auth: `CIRCLE_API_KEY` and `CIRCLE_ENTITY_SECRET`; wallet selection may use `CIRCLE_WALLET_SET_ID` or `CIRCLE_WALLET_SET_NAME`.
- Circle x402 Gateway / Agent Nanopayments - gateway operations exposed by `services/api/scripts/gateway.ts` through `createArcGatewayClient()`.
  - SDK/Client: `@circle-fin/x402-batching/client` in `services/api/src/integrations/circle.ts`.
  - Auth: `GATEWAY_PRIVATE_KEY`; optional RPC override: `ARC_RPC_URL`.
- Circle Agent Wallet CLI - shell commands generated by `services/api/scripts/circle-agent.ts` for wallets and spending-policy workflows.
  - SDK/Client: external Circle CLI, invoked through the local script rather than a Node SDK.
  - Auth: Circle CLI login/OTP outside this repository; policy inputs include `CIRCLE_AGENT_WALLET_ADDRESS`, `CIRCLE_POLICY_CHAIN`, and the `CIRCLE_POLICY_*` spend limits.
- Circle Gas Station / Paymaster - fail-closed, first-SCA-operation sponsor adapter in `services/api/src/integrations/paymaster.ts`, driven by `services/api/scripts/circle-paymaster.ts`.
  - SDK/Client: Circle developer-controlled-wallets SDK.
  - Auth: `CIRCLE_API_KEY`, `CIRCLE_ENTITY_SECRET`, `CIRCLE_PAYMASTER_ENABLED`, `CIRCLE_GAS_STATION_POLICY_CONFIRMED`, and `CIRCLE_PAYMASTER_*` allowlist/cap settings; the adapter additionally requires `STREAMING_VAULT_ADDRESS`.

## Data Storage

**Databases:**
- SQLite (local demo) - persisted in-process marketplace state via Node `DatabaseSync` in `services/api/src/persistence.ts` and created by `services/api/src/store.ts`.
  - Connection: `PACT_DB_PATH`; fallback path is `data/pact.sqlite` as documented in `README.md`.
  - Client: Node built-in `node:sqlite`; no ORM.
- SQLite (Circle paymaster ledger) - durable one-time sponsorship reservation ledger in `services/api/src/integrations/paymaster.ts`.
  - Connection: `CIRCLE_PAYMASTER_DB_PATH`, defaulting to `data/paymaster.sqlite`.
  - Client: Node built-in `node:sqlite`; the ledger uses WAL and full synchronous mode.
- PostgreSQL (optional production-style routes and indexer) - schema is `services/api/src/schema.sql`; `/pg` routes dynamically import repositories from `services/api/src/repositories/` in `services/api/src/app.ts`.
  - Connection: `DATABASE_URL`, with a local fallback in `services/api/src/db.ts` and `services/indexer/src/index.ts`.
  - Client: `pg` Pool; no ORM.

**File Storage:**
- Local filesystem only - SQLite files under `data/`, generated Solidity artifacts under `contracts/artifacts/`, static frontend build output in `frontend/dist/`, and local training datasets/outputs under `training/data/`.

**Caching:**
- Browser cache/query state: `@tanstack/react-query` in `frontend/src/App.tsx`.
- Server-side distributed cache: None detected.

## Authentication & Identity

**Auth Provider:**
- Custom bearer-token guard - API mutations use `PACT_AUTH_TOKEN` through `authGuard` in `services/api/src/security.ts` and `services/api/src/app.ts`; browser requests attach `VITE_API_TOKEN` in `frontend/src/api.ts`.
  - Implementation: constant-time Bearer token comparison; CORS parsing and protected mutation routes are configured in `services/api/src/security.ts` and `services/api/src/app.ts`.
- EVM wallet signature verification - task creation and production-style agent registration verify addresses/signatures using Viem in `services/api/src/app.ts`.
- Circle custody identity - optional Circle credentials support wallet creation and sponsorship; implementations are under `services/api/src/integrations/`.

## Environment Configuration

**Required env vars:**
- Local demo: none required; documented defaults in `README.md` run without wallets, external AI, Arc RPC, or Circle credentials.
- API hardening: `PACT_DB_PATH`, `PACT_AUTH_TOKEN`, `PACT_CORS_ORIGINS`, `PACT_ENABLE_DEMO_ENDPOINTS`, `PACT_HUMAN_REVIEWER_ID`, `PACT_RATE_LIMIT_WINDOW_MS`, `PACT_RATE_LIMIT_MAX`, `PACT_JSON_LIMIT`, and `PACT_ALLOW_UNSIGNED_TASKS`, read in `services/api/src/app.ts` and `services/api/src/store.ts`.
- OpenAI arbitration/agent operation: `OPENAI_API_KEY`, `ARBITRATOR_PROVIDER`, `ARBITRATOR_MODEL`, `ARBITRATOR_TIMEOUT_MS`, and `ARBITRATOR_FALLBACK`, consumed in `services/api/src/arbitration.ts` and `services/api/src/agent-runtime.ts`.
- PostgreSQL/indexer: `DATABASE_URL`, `RPC_URL`, and `VAULT_ADDRESS`, consumed in `services/api/src/db.ts` and `services/indexer/src/index.ts`.
- Arc deployment/Circle: `PRIVATE_KEY`, `ARC_RPC_URL`, `CIRCLE_API_KEY`, `CIRCLE_ENTITY_SECRET`, `CIRCLE_WALLET_SET_ID`, `CIRCLE_WALLET_SET_NAME`, `GATEWAY_PRIVATE_KEY`, plus the `CIRCLE_PAYMASTER_*` policy variables, consumed by `contracts/scripts/deploy_testnet.js` and `services/api/src/integrations/`.
- Browser build configuration: `VITE_API_URL`, `VITE_API_TOKEN`, `VITE_PACT_MODE`, and `VITE_AUTO_SEED_DEMO`, consumed by `frontend/src/api.ts` and `frontend/src/App.tsx`.

**Secrets location:**
- `.env.example` exists as a configuration template and `README.md` instructs copying it to `.env`; secret-file contents are not inspected. Runtime secrets are injected as process environment variables.

## Webhooks & Callbacks

**Incoming:**
- None detected. HTTP API routes are conventional Express REST endpoints in `services/api/src/app.ts`, not third-party webhook receivers.
- EVM event callbacks are local RPC subscriptions: `StreamCreated`, `StreamPaused`, and `CollateralSlashed` are watched in `services/indexer/src/index.ts` and translated into PostgreSQL task updates.

**Outgoing:**
- OpenAI API calls originate in `services/api/src/agent-runtime.ts` and `services/api/src/arbitration.ts` only when live provider configuration is enabled.
- Circle wallet, x402 gateway, and gas-sponsorship calls originate in `services/api/src/integrations/circle.ts`, `services/api/src/integrations/paymaster.ts`, and their scripts in `services/api/scripts/`.
- WebSocket stream updates originate from `services/api/src/server.ts` and are consumed by the browser in `frontend/src/App.tsx`; this is a first-party callback channel, not an external webhook.
